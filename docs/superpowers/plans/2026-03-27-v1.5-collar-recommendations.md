# Options Trader v1.5 — Collar Recommendations (PRD)

> **Status:** Draft — awaiting owner approval before implementation begins.

---

## Background & Motivation

A **collar** is a protective options strategy used to limit downside risk on a large equity holding without incurring net cost. It combines:

- **Covered Call (CC):** Sell an OTM call against shares already owned. Collected premium funds the hedge.
- **Long Put (LP):** Buy an OTM put to establish a price floor. Cost is offset (ideally fully) by the CC premium.

The net result is a zero-cost (or near-zero-cost) hedge: the investor caps upside at the call strike in exchange for guaranteed downside protection below the put strike.

This strategy is most valuable on **concentrated large positions** — where a sudden market move could cause significant dollar loss.

---

## Goal

Add a new **Collar Mode** to Options Trader that:

1. Identifies eligible equity holdings (>$50,000 current market value).
2. Finds qualifying collar pairs (covered call + long put) that are self-financing — i.e., CC premium ≥ LP cost.
3. Deduplicates and ranks recommendations to surface the best option per calendar month per stock.
4. Delivers recommendations weekly via email every **Saturday at 7:00 AM PST**.

## Eligible Holdings

A holding qualifies for collar analysis if:

```
current_market_value = shares_owned × current_price > $50,000
```

Source: same Robinhood portfolio snapshot used by the daily pipeline.

---

## Option Chain Filtering

For each eligible holding, scan the full options chain and apply the following filters to build candidate pairs:

### Expiration Window

```
4 weeks ≤ DTE ≤ 16 weeks   (i.e., 28 ≤ DTE ≤ 112 calendar days)
```

### Covered Call Leg Criteria

| Criterion | Rule |
|---|---|
| Type | Call |
| Strike | ≥ current_price × 1.10  (at least 10% OTM) |
| Expiration | Within the 4–16 week window |
| Open Interest | > 5 (i.e., ≥ 6) |

### Long Put Leg Criteria

| Criterion | Rule |
|---|---|
| Type | Put |
| Strike | ≥ current_price × 0.90  (no more than 10% below current price) |
| Expiration | **Exact same expiration date** as the paired call |
| Open Interest | > 5 (i.e., ≥ 6) |

### Self-Financing Criterion (Hard Gate)

```
CC mid price ≥ LP mid price
AND
CC mid − LP mid ≥ $0.10   (minimum net gain per share)
```

Only pairs that fully cover the put cost **and** yield at least $0.10/share net pass this gate. Net gain = CC mid − LP mid (per share, × 100 per contract).

---

## Collar Pair Construction

For each eligible holding:

1. Enumerate all (CC, LP) pairs where both legs share the **exact same expiration date**, both pass the individual leg criteria, and the self-financing gate is met (net gain ≥ $0.10/share).
2. For pairs sharing the same calendar expiration month — keep only the **one with the highest net gain** (CC mid − LP mid). This is the deduplication step.
3. Result: at most **one qualifying collar recommendation per calendar month** per stock.
4. Return all qualifying months — multiple months are allowed, but only one rec per month.

### Contracts

Number of contracts = `floor(shares_owned / 100)`. The same count applies to **both** CC and LP legs — a collar is always a 1-to-1 mapping (N covered calls paired with N long puts).

### Fallback: No Qualifying Collars for a Stock

If a stock has eligible holdings (market value > $50,000) but **zero pairs pass the $0.10/share net gain floor**, the pipeline applies a relaxed fallback:

1. Re-scan all (CC, LP) pairs for that stock using the same expiration-date, OTM%, and OI criteria — but **dropping** the $0.10/share net gain floor.
2. Among any self-financing pairs (CC mid ≥ LP mid, net gain ≥ $0.00), return the single best pair (highest net gain), ignoring the per-month deduplication.
3. Mark this recommendation as **`low_gain: true`** in the data structure.
4. Render the recommendation's two-row tuple with a **light red background** in the email, with a note: `⚠ Best available — below $0.10/share threshold`.

If even this relaxed scan finds no self-financing pair (CC mid < LP mid for all combinations), the stock is **omitted from the report entirely**.

| Level | Rule |
|---|---|
| Within a month | Keep only the pair with highest net gain (CC mid − LP mid) |
| Across months | Allow up to one recommendation per calendar month |
| Across symbols | Each symbol gets its own group; groups are independent |

## Email Report

### Schedule

- **Frequency:** Once per week
- **Day/Time:** Saturday at 7:00 AM PST (converted to local scheduler time, DST-aware)
- **Recipient:** Same as existing daily report (`recipient_email` in config)
- **Subject line:** `[Collar Report] Options Trader — {date}`

### Report Structure

The collar report uses the same visual design as the existing covered-call report:
- Same header/branding
- Same summary bar (adapted fields — see below)
- Same table layout

**Summary bar fields:**

| Field | Description |
|---|---|
| Eligible Holdings | Count of stocks with >$50K market value analyzed |
| Collar Opportunities | Total recommendation count (across all symbols and months) |
| Symbols | Count of distinct symbols with at least one qualifying collar |
| Below Threshold | Count of fallback (low-gain) recommendations shown in light red |
| Earnings Flags | Count of collars where an earnings date falls in the expiration month |

### Table Layout

Recommendations are grouped by **symbol (equity bucket)**, ordered alphabetically. Within each symbol group, recommendations are ordered by **expiration month** (ascending).

Each collar recommendation occupies **two consecutive rows** forming a tuple:

| Row | Leg | Description |
|---|---|---|
| Row 1 | Covered Call | Strike, expiration, DTE, bid, ask, mid, OTM%, Ann.Yield, YPD/day, OI |
| Row 2 | Long Put | Strike, expiration, DTE, bid, ask, mid, protection%, cost, YPD equiv., OI |

**Combined cell** (spans both rows, rightmost column):

| Field | Value |
|---|---|
| Net Gain | `(CC mid − LP mid) × 100 × contracts` |
| Net Gain/share | `CC mid − LP mid` |
| Protection Floor | `put_strike / current_price − 1` (e.g., −8.5%) |
| Upside Cap | `call_strike / current_price − 1` (e.g., +12.0%) |
| Contracts | Number of contracts |
| Expiration Month | e.g., `May 2026` |

**Low-gain fallback styling:** When `low_gain: true`, both rows of the tuple receive a light red background (`#fff1f2` / `#ffe4e6`) and a note row below:

```
⚠ Best available — below $0.10/share threshold
```

**Earnings note row** (conditional, spans full width, appears below the two-row tuple if applicable):

```
⚠ Earnings: 2026-05-14
```

**Symbol group separator:** A shaded row showing `SYMBOL — CompanyName — $XXX,XXX market value — N contracts` spanning all columns, above each symbol's recommendations.

## Intraday Direction Filter (v1.6+)

Recommendations are post-filtered based on the stock's intraday price direction
(current price vs. previous session close) before the email is composed.
The rationale: each strategy has better entry conditions depending on the day's movement.

| Strategy | Filter | Rationale |
|---|---|---|
| **Collar** | Stock trading **UP** | Lock in a higher call-strike and buy downside protection while the position is at a relative peak. |
| **CCS** (Bear Call Spread) | Stock trading **UP** | Enter a bearish spread when the stock has moved higher, giving the short call more cushion. |
| **PCS** (Bull Put Spread) | Stock trading **DOWN** | Enter a bullish spread when the stock has pulled back, giving the short put more cushion. |

Direction is determined via yfinance `fast_info` (`last_price` vs `previous_close`).

**Fail-open policy:** If direction cannot be determined (`unknown`) or the stock is `flat`,
the recommendation passes through and is **not** filtered. This prevents a data hiccup
from silently dropping valid recommendations.

## Acceptance Criteria

1. Only holdings with market value > $50,000 appear in the collar report.
2. All CC legs are ≥ 10% OTM; all LP legs are within 10% OTM.
3. Both legs of every collar pair share the **exact same expiration date**.
4. Both legs have OI > 5 (≥ 6).
5. All standard recommendations are self-financing with net gain ≥ $0.10/share.
6. CC and LP leg contract counts are always equal (1-to-1 mapping).
7. At most one qualifying collar is recommended per symbol per calendar month.
8. Fallback (low-gain) recs are rendered in light red with a threshold warning note; they appear only when no standard qualifying pair exists for a stock.
9. A stock is omitted entirely only when no self-financing pair (net gain ≥ $0.00) can be found.
10. Earnings dates within the expiration month are surfaced on the relevant recommendation.
11. Email is sent every weekday at 10:30 AM ET regardless of whether qualifying collars are found.
12. `--collar-dry-run` generates the email HTML locally without sending.
13. Existing covered-call pipeline and daily schedule are unaffected.
14. CCS recommendations are only included when the underlying stock is trading UP (or direction is unknown/flat).
15. PCS recommendations are only included when the underlying stock is trading DOWN (or direction is unknown/flat).
16. Collar recommendations are only included when the underlying stock is trading UP (or direction is unknown/flat).
17. A failed direction lookup never silently drops a recommendation (fail-open).
