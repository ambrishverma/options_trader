# Options Trader v1.5 ŌĆö Collar Recommendations (PRD)

> **Status:** Draft ŌĆö awaiting owner approval before implementation begins.

## Background & Motivation

A **collar** is a protective options strategy used to limit downside risk on a large equity holding without incurring net cost. It combines:

- **Covered Call (CC):** Sell an OTM call against shares already owned. Collected premium funds the hedge.
- **Long Put (LP):** Buy an OTM put to establish a price floor. Cost is offset (ideally fully) by the CC premium.

The net result is a zero-cost (or near-zero-cost) hedge: the investor caps upside at the call strike in exchange for guaranteed downside protection below the put strike.

This strategy is most valuable on **concentrated large positions** ŌĆö where a sudden market move could cause significant dollar loss.

---

## Goal

Add a new **Collar Mode** to Options Trader that:

1. Identifies eligible equity holdings (>$50,000 current market value).
2. Finds qualifying collar pairs (covered call + long put) that are self-financing ŌĆö i.e., CC premium Ōēź LP cost.
3. Deduplicates and ranks recommendations to surface the best option per calendar month per stock.
4. Delivers recommendations weekly via email every **Saturday at 7:00 AM PST**.

## Option Chain Filtering

For each eligible holding, scan the full options chain and apply the following filters to build candidate pairs:

### Expiration Window

```
4 weeks Ōēż DTE Ōēż 16 weeks   (i.e., 28 Ōēż DTE Ōēż 112 calendar days)
```

### Covered Call Leg Criteria

| Criterion | Rule |
|---|---|
| Type | Call |
| Strike | Ōēź current_price ├Ś 1.10  (at least 10% OTM) |
| Expiration | Within the 4ŌĆō16 week window |
| Open Interest | > 5 (i.e., Ōēź 6) |

### Long Put Leg Criteria

| Criterion | Rule |
|---|---|
| Type | Put |
| Strike | Ōēź current_price ├Ś 0.90  (no more than 10% below current price) |
| Expiration | **Exact same expiration date** as the paired call |
| Open Interest | > 5 (i.e., Ōēź 6) |

### Self-Financing Criterion (Hard Gate)

```
CC mid price Ōēź LP mid price
AND
CC mid ŌłÆ LP mid Ōēź $0.10   (minimum net gain per share)
```

Only pairs that fully cover the put cost **and** yield at least $0.10/share net pass this gate. Net gain = CC mid ŌłÆ LP mid (per share, ├Ś 100 per contract).

---

## Collar Pair Construction

For each eligible holding:

1. Enumerate all (CC, LP) pairs where both legs share the **exact same expiration date**, both pass the individual leg criteria, and the self-financing gate is met (net gain Ōēź $0.10/share).
2. For pairs sharing the same calendar expiration month ŌĆö keep only the **one with the highest net gain** (CC mid ŌłÆ LP mid). This is the deduplication step.
3. Result: at most **one qualifying collar recommendation per calendar month** per stock.
4. Return all qualifying months ŌĆö multiple months are allowed, but only one rec per month.

### Contracts

Number of contracts = `floor(shares_owned / 100)`. The same count applies to **both** CC and LP legs ŌĆö a collar is always a 1-to-1 mapping (N covered calls paired with N long puts).

### Fallback: No Qualifying Collars for a Stock

If a stock has eligible holdings (market value > $50,000) but **zero pairs pass the $0.10/share net gain floor**, the pipeline applies a relaxed fallback:

1. Re-scan all (CC, LP) pairs for that stock using the same expiration-date, OTM%, and OI criteria ŌĆö but **dropping** the $0.10/share net gain floor.
2. Among any self-financing pairs (CC mid Ōēź LP mid, net gain Ōēź $0.00), return the single best pair (highest net gain), ignoring the per-month deduplication.
3. Mark this recommendation as **`low_gain: true`** in the data structure.
4. Render the recommendation's two-row tuple with a **light red background** in the email, with a note: `ŌÜĀ Best available ŌĆö below $0.10/share threshold`.

If even this relaxed scan finds no self-financing pair (CC mid < LP mid for all combinations), the stock is **omitted from the report entirely**.

## Intraday Direction Filter (v1.6+)

Recommendations are post-filtered based on the stock's intraday price direction
(current price vs. previous session close) before the email is composed.
The rationale: each strategy has better entry conditions depending on the day's movement.

| Strategy | Filter | Rationale |
|---|---|---|
| **Collar** | Stock trading **UP** | Lock in a higher call-strike and buy downside protection while the position is at a relative peak. |
| **CCS** (Bear Call Spread) | Stock trading **UP** | Enter a bearish spread when the stock has moved higher, giving the short call more cushion. |
| **PCS** (Bull Put Spread) | Stock trading **DOWN** | Enter a bullish spread when the stock has pulled back, giving the short put more cushion. |

Direction is determined via yfinance `fast_info` (`last_price` vs `previous_close`).

**Fail-open policy:** If direction cannot be determined (`unknown`) or the stock is `flat`,
the recommendation passes through and is **not** filtered. This prevents a data hiccup
from silently dropping valid recommendations.

## Panic Roll 7-Day Minimum Expiration (v1.7.2)

### Problem

Panic mode was rolling covered-call contracts to the very next available expiration, which could
be only days away ŌĆö leaving almost no time value and no meaningful premium recovery.

### Change

`trader.py` ŌĆö `_execute_panic_roll()` now computes `min_exp_date = today + 7 days` and filters
`future_exps` to only expirations that are **both** after the current contract's expiration and at
least 7 calendar days from today.

```python
min_exp_date = (date.today() + timedelta(days=7)).isoformat()
future_exps = [
    e for e in all_expirations
    if e > expiration and e >= min_exp_date
]
```

If no such expiration exists the roll result carries an error:
`"No future expirations available beyond {expiration} that are at least 7 days out"`

### Acceptance Criteria (v1.7.2)

30. Panic roll never targets an expiration within 6 calendar days of today.
31. If only near-term expirations are available the roll result contains an appropriate error message and no order is placed.

## Collar CLI Refactor (v1.8)

### Overview

The `--collar-run SYMBOL` command has been replaced with a unified `--collar [SYMBOL]` command
that accepts sub-options to control behavior.

### New CLI Syntax

```
python main.py --collar [SYMBOL] [--find] [--add] [--show] [--roll] [--weeks N [N]] [--prompt]
```

| Invocation | Behavior |
|---|---|
| `--collar` (no symbol) | Run full Saturday collar pipeline (same as before) |
| `--collar TSLA --find` | On-demand collar scan for TSLA (replaces `--collar-run TSLA`) |
| `--collar TSLA --add` | Place the top-ranked collar order for TSLA (prompt by default) |
| `--collar [TSLA] --show` | Show existing collar holdings; optionally filter by symbol |
| `--collar TSLA --roll` | Find roll-up / roll-out options for existing TSLA collar |
| `--weeks 4 26` | Override DTE window for `--find` or `--roll` (default `4 26`) |
| `--prompt` | Require y/n confirmation before placing any order |

### Sub-option Details

#### `--find`

Calls `run_collar_on_demand(symbol, dte_min, dte_max)` from `collar.py` and prints the
recommendation table to the console. Identical to the old `--collar-run` behavior.

#### `--add`

1. Calls `run_collar_on_demand(symbol, dte_min, dte_max)` to get the top recommendation.
2. Calls `place_collar_order(symbol, rec, prompt=args.prompt)` from `trader.py`.
3. `place_collar_order` places two sequential Robinhood orders:
   - **STO call:** `robin_stocks.orders.order_sell_option_limit()` (covered call, `positionEffect="open"`)
   - **BTO put:** `robin_stocks.orders.order_buy_option_limit()` (long put, `positionEffect="open"`)
4. If the STO call fails, the BTO put is not attempted.

#### `--show`

Calls `show_collar_holdings(symbol=None)` from `trader.py`.
- Logs into Robinhood, fetches all open option positions.
- Groups by `(symbol, expiration)` and identifies collar pairs: short CALL + long PUT.
- Prints a table showing call strike, put strike, quantity, average prices, and net/share.
- Unpaired legs (call without matching put or vice versa) are listed separately.
- If no symbol is provided, shows all collar holdings across the portfolio.

#### `--roll`

1. Logs into Robinhood to fetch current collar positions for the symbol.
2. Determines `min_call_strike` (highest short call) and `min_put_strike` (highest long put)
   from existing positions.
3. Calls `find_collar_roll_options(symbol, dte_min, dte_max, min_call_strike, min_put_strike)`
   from `collar.py`.
4. `find_collar_roll_options` wraps `run_collar_on_demand` and post-filters recommendations to
   require `call_leg.strike >= min_call_strike` AND `put_leg.strike >= min_put_strike`.
5. If no higher-strike collars are found, falls back to all recommendations and sets
   `roll_constrained: True` in the result dict.
6. Prints the roll options table to the console.

### New Module Functions

| Module | Function | Description |
|---|---|---|
| `trader.py` | `show_collar_holdings(symbol=None)` | Display collar pairs from Robinhood positions |
| `trader.py` | `place_collar_order(symbol, rec, prompt=True) -> bool` | Place STO call + BTO put orders |
| `collar.py` | `find_collar_roll_options(symbol, dte_min, dte_max, min_call_strike=0.0, min_put_strike=0.0) -> dict` | Find roll-up/roll-out collar options |

### Removed

- `--collar-run SYMBOL` CLI flag (replaced by `--collar SYMBOL --find`)
- `cmd_collar_run()` function in `main.py`

## v1.9 ŌĆö PCS / CCS Sub-option Refactor (Apr 11, 2026)

### Motivation

The `--pcs` and `--ccs` commands previously only supported a single action (scan / find).
This update adds the same `--find / --add / --show / --close` sub-option pattern introduced
for `--collar` in v1.8, so traders can manage full spread lifecycle from the CLI.

| `--pcs --show` | Show all open PCS (Put Credit Spread) holdings across the portfolio |
| `--pcs SYMBOL --show` | Show PCS holdings for SYMBOL only |
| `--pcs SYMBOL --close` | Close the existing PCS position for SYMBOL at current mid-price |
| `--ccs SYMBOL` | Find best CCS for SYMBOL (default action = `--find`) |
| `--ccs SYMBOL --find` | Explicit find |
| `--ccs SYMBOL --add` | Find best CCS for SYMBOL and interactively place the order |
| `--ccs --show` | Show all open CCS (Call Credit Spread) holdings across the portfolio |
| `--ccs SYMBOL --show` | Show CCS holdings for SYMBOL only |
| `--ccs SYMBOL --close` | Close the existing CCS position for SYMBOL at current mid-price |

### New Module Functions

| Module | Function | Description |
|---|---|---|
| `trader.py` | `show_spread_holdings(spread_type, symbol=None)` | Display PCS or CCS pairs from Robinhood positions |
| `trader.py` | `place_spread_order(symbol, rec, spread_type, prompt=True) -> bool` | Place new PCS or CCS spread order via `order_option_spread` |
| `trader.py` | `close_spread_position(symbol, spread_type, prompt=True) -> bool` | Close an open PCS or CCS spread at current mid net debit |
| `main.py` | `cmd_pcs_find/add/show/close()` | Dispatcher functions for each PCS sub-option |
| `main.py` | `cmd_ccs_find/add/show/close()` | Dispatcher functions for each CCS sub-option |

### Order Placement Details

**`--add` (open new spread)**
- Calls `scan_pcs` / `scan_ccs` to find the top recommendation.
- Calls `place_spread_order(symbol, rec, spread_type="PCS"/"CCS", prompt=True)`.
- Uses `robin_stocks.orders.order_option_spread` with `direction="credit"`.
- Spread legs: short leg `action="sell" / effect="open"`, long leg `action="buy" / effect="open"`.
- Strike prices formatted as `f"{strike:.4f}"` (4dp) to match Robinhood's stored format.
- Always prompts for y/n confirmation before submitting.

**`--close` (close existing spread)**
- Fetches open option positions from Robinhood and identifies the spread pair.
- Fetches live bid/ask for each leg via `get_option_market_data_by_id`.
- Closing net debit = `short_mid ŌłÆ long_mid` (cost to buy back short, less credit for selling long).
- Calls `order_option_spread` with `direction="debit"` (or `"credit"` if spread has fully decayed).
- Spread legs: short leg `action="buy" / effect="close"`, long leg `action="sell" / effect="close"`.
- Always prompts for y/n confirmation before submitting.

## v2.1 ŌĆö Optimize Mode (Apr 11, 2026)

### Overview

Adds a fourth automated protection mode ŌĆö **Optimize mode** ŌĆö that runs FIRST in the daily pipeline (before safety, rescue, and panic). Its goal is to improve open short option positions that have become expensive (gained >40% vs. original credit) by rolling to a better strike (UP for CALLs, DOWN for PUTs) or a later expiration within a 10-day window.

**Pipeline sequence (updated):** optimize ŌåÆ safety ŌåÆ rescue ŌåÆ panic

### Trigger Condition

For each short option contract with DTE Ōēź 1:

```
current_mid >= abs(purchase_price) / 100 ├Ś 1.40
```

Where `purchase_price` is the raw Robinhood `average_price` (negative for short/credit-received). Contracts with `purchase_price == 0` or `current_mid == 0` are silently skipped.

### Roll Target Selection

- **Expiration window**: current expiration through current expiration + 10 calendar days (inclusive)
- **Strike direction**: CALL ŌåÆ strike Ōēź current (raise ceiling); PUT ŌåÆ strike Ōēż current (lower floor)
- **Selection metric**: maximise Risk/Reward ratio among credit-positive candidates
  - Reward = `STO_mid ŌłÆ BTC_mid`
  - Risk = `|new_strike ŌłÆ live_price|` (floored at ╬Ą = 0.001 to avoid ├Ę0)
  - R/R = Reward / Risk
- If no credit-positive candidate exists across all scanned expirations: `skipped=True`, no order placed

### Execution

1. Cancel ALL outstanding orders for the contract
2. Wait **20 seconds** (only if any orders were cancelled)
3. Submit atomic roll-forward spread at net mid credit (GTC limit)

### Key Differences vs Rescue Mode

| Concern | Rescue | Optimize |
|---|---|---|
| Trigger | ITM at DTE 1-2 | Option price gained >40% |
| Expiration window | Next available Ōēź 7 days | Current exp + up to 10 days |
| Wait after cancel | 30s | 20s |
| Aborts if no credit | Yes (skipped) | Yes (skipped) |
| DTE filter | 1-2 only | Any Ōēź 1 |

### Contracts Excluded from Downstream Modes

Successfully acted-on optimize contracts (`success=True`) are excluded from safety, rescue, and panic modes. Skipped optimize contracts (no credit found) remain eligible for all downstream modes.

### New Module: `trader.py` ŌĆö `execute_optimize_rolls`

```
execute_optimize_rolls(open_short_contracts, live_prices, name_map, dry_run)
```

Returns list of result dicts. Each dict contains:
`symbol, name, opt_type, strike, expiration, dte, next_expiration, target_strike, next_dte, quantity, live_price, purchase_price_per_share, current_mid, gain_pct, orders_cancelled, success, skipped, order_id, net_price, direction, net_label, error`
